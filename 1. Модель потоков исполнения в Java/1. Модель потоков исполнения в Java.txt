~~~~~~~~~~~~~~~~~~~~~МОДЕЛЬ ПОТОКОВ ИСПОЛНЕНИЯ В JAVA~~~~~~~~~~~~~~~~~~

 Потоки исполнения обеспечивают асинхронность работы
исполняющей среды Java
 Работу многопоточности можно понять в сравнении:
 1. Однопоточная система.
    Тут принимается /цикл ожидания событий с опросом/
    Поток управления один и он в бесконечном цикле опрашивает одну
    очередь событий 
    Если возвращается сигнал готовности файла к чтению, цикл ожидания
    передает управление соотв. обработчику событий.
    Во время выполнения в программе ничего не может произойти
    Т.о. В однопоточной системе поток выполнения блокируется по 
    причине ожидание некого ресурса.
 2. Многопоточная система.
    Тут один поток может быть приостановлен без остановки всех
    остальных частей программы
    Так во время ожидания при чтении файла можно использовать в
    другом месте программы

 Потоки выполнения находятся в нескольких состояниях:
 1. Поток /выполняется/		- понятно
 2. Поток /готов к выполнению/	- он получил время ЦП
 3. Поток /приостановлен/	- он временно прекратил активность
 4. Поток /возобновлен/		- он продолжает работу с того места,
				  где остановился
 5. Поток /заблокирован/	- он ждёт какой-то ресурс
 6. Поток /прерван/		- он полностью остановил выполнение
